 Features & Highlights
Problem:
Olist had massive raw transactional data across 7 tables with no unified view of revenue performance, customer geography, product demand, or payment behavior.
Key Goal:
Build a structured MySQL database from scratch, clean and import 100K+ records, and answer 6 critical business questions using pure SQL — no BI tools, no Python.
